方法（可解釋、可溝通）
1) 基礎可達性分數
以館點周邊 500m / 1000m 緩衝區計算站點密度，依交通模式與距離權重加總為 0–100 分；分數高代表「更容易靠公共運輸抵達」。
2) Access deserts
用格網掃描城市範圍，估計每個 cell 的「最佳可達性」並與門檻比較，標示出服務落差熱點（desert cells）。
3) 外展點位建議
在 deserts 周邊依「能覆蓋落差」與「場址本身可達性」打分，產出 Top N 候選點位，支援場域合作與政策溝通。
如何用它做簡報？
你可以先用控制台做 what-if，系統會把「參數假設 → 指標影響」轉成可讀的摘要，並支援用 URL 分享同一組假設到「一頁簡報 / 成果頁」。
